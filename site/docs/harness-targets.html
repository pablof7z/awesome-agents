Harness targets

Where profiles land for each harness, and how to run them.

Project installs

Project-level installs write into the current repo:

HarnessPath
claude-code.claude/agents/<profile>.md
opencode.opencode/agents/<profile>.md
codexNot supported; loads from user config.
tenex-edgeNot supported; agents are machine-local.

Global installs

HarnessPath
codex$CODEX_HOME/<profile>.config.toml
claude-code$CLAUDE_HOME/agents/<profile>.md
opencode$OPENCODE_CONFIG_DIR/agents/<profile>.md
tenex-edge$TENEX_EDGE_HOME/agents/<profile>.json

Each falls back to a sensible home (~/.codex, ~/.claude, ~/.config/opencode, ~/.tenex-edge) when the env var is unset.

Running installed profiles

After an install, the CLI prints the run command for each harness CLI it finds on PATH:

$ codex --profile <profile>
$ claude --agent <profile>
$ tenex-edge launch <profile>

Codex expects a plain profile name, not a path passed to --profile.
