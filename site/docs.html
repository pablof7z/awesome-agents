Install a whole agent, from any source.

awesome-agents is an npx-compatible installer for reusable agent profiles. It mirrors the useful parts of npx skills, but the unit is a whole operational agent profile, not a skill: its prompt, model, and tool boundaries. Sources are GitHub repos, Git URLs, or local checkouts.

Overview

Profiles are read from YAML or Markdown files under agents/<slug>/, adapted for the selected harness, and installed into the right place for Codex, Claude Code, OpenCode, or tenex-edge. YAML profile files are preferred; the loader also accepts Markdown files with YAML frontmatter for compatibility with .agent.md-style profiles.

The command shape follows npx skills: add to install, use to render without installing, plus list, remove, update, and init. Everything is fully scriptable: human output uses subtle ANSI color, and --json gives you a machine-readable form.

Install & run

Run it straight from npm with npx:

$ npx awesome-agents add owner/repo --agent triage-agent
$ npx awesome-agents add owner/repo --agent triage-agent --harness tenex-edge

From a checkout of this repo during development:

$ npm install
$ npm test
$ node ./bin/awesome-agents.js add ./test/fixtures/profile-source --agent triage-agent --dry-run

Commands

CommandWhat it does
add <source>Install a profile from a source package.
install <source>Alias for add.
use <source[@profile]>Render a single profile without installing it.
list / lsShow installed profiles.
remove <profile…> / rmRemove installed profiles.
update [profile…] / upgradeReinstall from the recorded source.
init [name]Create a profile-source skeleton.

Install options

  • --agent <slug> selects an agent profile, e.g. --agent triage-agent.
  • --profile <slug> or --skill <slug> are explicit profile aliases. --skill is command-shape compatibility only; it does not mean the artifact is a skill.
  • Omit a profile selector in an interactive terminal to choose from a checkbox list; every profile is selected by default.
  • --yes accepts detected profile and harness selections without opening selectors.
  • --harness codex|claude-code|opencode|tenex-edge|* selects target harnesses. Without it, the CLI detects harness CLIs on PATH; if none are detected, pass --harness explicitly.
  • --all installs every profile to every supported harness.
  • --dry-run previews writes without touching disk.
  • --project installs at project level where supported; --global installs at user level. Codex and tenex-edge always install globally.
  • --list inspects available source profiles without installing.
  • --json for machine-readable output; NO_COLOR=1 disables color.

Harness targets

Project installs write to:

  • Codex  not supported; Codex profiles load from user config.
  • Claude Code  .claude/agents/<profile>.md
  • OpenCode  .opencode/agents/<profile>.md
  • tenex-edge  not supported; tenex-edge agents are machine-local.

Global installs write to:

  • Codex  $CODEX_HOME/<profile>.config.toml, or ~/.codex/<profile>.config.toml
  • Claude Code  $CLAUDE_HOME/agents/<profile>.md, or ~/.claude/agents/<profile>.md
  • OpenCode  $OPENCODE_CONFIG_DIR/agents/<profile>.md, or ~/.config/opencode/agents/<profile>.md
  • tenex-edge  $TENEX_EDGE_HOME/agents/<profile>.json, or ~/.tenex-edge/agents/<profile>.json

After an install, the CLI groups run commands by profile for the harness CLIs it finds on PATH:

$ codex --profile <profile>
$ claude --agent <profile>
$ tenex-edge launch <profile>

Codex expects a plain profile name, not a path passed to --profile.

Source format

An agent-profile source should look like this:

agents/
  triage-agent/
    agent.yaml
  ops-agent/
    agent.agf.yaml
    skills/
      gh-pages-publisher/
        SKILL.md
    scripts/
      heartbeat.sh
    references/
      runbook.md

YAML profile files are preferred. Agent-owned scripts/ and references/ are installed into ~/.agents/homes/<slug>/scripts and ~/.agents/homes/<slug>/references.

Declaring skills

Profiles may declare immediately relevant skills:

skills:
  - gh-pages-publisher
  - pablof7z/tenex-edge basic-skill

Bare skill names are resolved from the profile directory, then the source checkout, then ~/.agents/skills. Source-qualified entries use the same source-plus-skill selector shape as npx skills add <source> --skill <skill>. Declared skills are copied into ~/.agents/homes/<slug>/skills/<skill> and listed with complete paths in the installed agent prompt.

Examples

$ npx awesome-agents add owner/repo --list
$ npx awesome-agents add owner/repo --agent triage-agent
$ npx awesome-agents add owner/repo --agent triage-agent --harness codex --global
$ npx awesome-agents add owner/repo --agent triage-agent --harness tenex-edge
$ npx awesome-agents add owner/repo --all --dry-run
$ npx awesome-agents use owner/repo --agent triage-agent --harness claude-code
$ npx awesome-agents list --json
$ npx awesome-agents remove triage-agent --agent codex
$ npx awesome-agents update triage-agent --agent codex --dry-run

Registry & safety

The CLI keeps its own registry at .awesome-agents/installed.json for project installs, or ~/.awesome-agents/installed.json for global installs. list, remove, and update use this registry.

Generated files are marked Generated by awesome-agents. The CLI refuses to overwrite or delete any file that does not contain that marker unless you pass --force, so it never clobbers something it did not write.
